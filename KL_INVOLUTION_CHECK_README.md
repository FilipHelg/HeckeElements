# kl_involution_check - Kazhdan-Lusztig Involution Analysis Tool

## Overview

`kl_involution_check` is a specialized analysis tool for studying Kazhdan-Lusztig (KL) polynomials in the symmetric group S_n. It focuses on a specific involution `w` and identifies pairs of elements `(x, y)` that share the same left cell (have identical P-tableaux) and satisfy the property that:

```
dKH_w * kH_x = dKH_w * kH_y (non-zero)
```

Where:
- `dKH_w` is the dual KL base element for the involution w
- `kH_x` is the KL base element for x
- The product is computed in the Hecke algebra

## Compilation

The program is included in the makefile and can be compiled with:

```bash
make kl_involution_check
```

Or compile directly:
```bash
gcc -std=c11 -Ofast -march=native -Wall -fopenmp -c kl_involution_check.c -o kl_involution_check.o
gcc -o kl_involution_check kl_involution_check.o lehmer.o -lm -fopenmp
```

## Usage

```
kl_involution_check n w [OPTIONS]
```

### Required Arguments

- `n`: The size of the symmetric group S_n (must be between 3 and 8)
- `w`: The Lehmer index of an involution in S_n

### Optional Arguments

- `--dual-bin PATH`: Path to the dual KL polynomial cache (default: `dual_kl_bulk_n<n>.bin`)
- `--kl-data PATH`: Path to the KL polynomial data (default: `S<n>.txt`)
- `--out PATH`: Path for the output CSV file (default: `kl_involution_n<n>_w<w>.csv`)

## Examples

### Example 1: Analyze an involution in S_5

```bash
./kl_involution_check 5 42
```

This analyzes involution with Lehmer index 42 in S_5 and generates:
- Terminal output with a summary of results
- CSV file `kl_involution_n5_w42.csv` with the pairs

### Example 2: Use custom paths

```bash
./kl_involution_check 6 100 \
  --dual-bin ./my_dual_cache.bin \
  --kl-data ./my_kl_data.txt \
  --out ./results.csv
```

## Input Requirements

### Lehmer Index
The program accepts Lehmer indices (0 to n!-1) for elements of S_n. The program **automatically verifies** that the provided index corresponds to an involution (an element w such that w*w = identity).

### Involution Validation
Before processing, the program checks:
1. The input w is a valid Lehmer index for S_n
2. w is an involution (w*w = identity in the group)

If the validation fails, the program will:
- Report an error message to stderr
- Show the permutation representation of w
- Exit with code 1

Example error output:
```
Error: w=42 is not an involution in S_5.
Permutation: 0 2 3 1 4
```

### Required Data Files
The program requires:
- **Dual KL cache** (`dual_kl_bulk_n<n>.bin`): Binary cache of dual KL polynomials for involutions
- **KL data** (`S<n>.txt`): Text file with KL polynomial data (for n >= 4)

These files are typically generated by `dual_kl_bulk` and `kl_structure_bulk_opt_parallel`.

## Output

### Terminal Output
The program prints a summary to the terminal:

```
========================================
Involution Analysis Complete
========================================
S_5, w=42: 0 2 3 1 4

Total left-cell buckets processed: 156
Non-empty buckets (size > 1): 47
Pairs found where dKH_w*kH_x = dKH_w*kH_y (non-zero): 523

```

### CSV Output
The output CSV file has the following format:

```csv
w,x,y,w_perm,x_perm,y_perm
42,15,32,0 2 3 1 4,0 1 4 3 2,0 3 1 2 4
42,15,89,0 2 3 1 4,0 1 4 3 2,1 0 2 3 4
...
```

Each row represents:
- `w`: The involution index
- `x, y`: Indices of elements in the same left cell
- `w_perm, x_perm, y_perm`: The permutation representations (space-separated)

The pairs (x,y) are ordered so that x < y.

## Algorithm Overview

1. **Validation**: Verify that w is an involution
2. **Context Setup**: Build all necessary lookup tables for S_n
3. **Data Loading**: Load dual KL polynomials and KL data
4. **Left Cell Grouping**: Identify all left cells by grouping elements with identical P-tableaux
5. **Product Computation**: For each left cell with more than one element:
   - Compute dKH_w * kH_x for each x in the cell
   - Compare products to find matching pairs
   - Write matching pairs to output
6. **Summary**: Report statistics on buckets and pairs found

## Performance Characteristics

- **Memory**: Scales with n! (factorial of n), typically manageable for n ≤ 8
- **Time**: Depends on the number of non-empty left cells and the complexity of polynomial products
- **Parallelization**: Uses OpenMP for parallel processing (inherited from the KL computation framework)

## Technical Notes

### Left Cells vs Two-Sided Cells
The program uses **left cells** (defined by identical P-tableaux) rather than two-sided cells. This distinction is important for the mathematical interpretation of the results.

### Involution Definition
An involution is an element w ∈ S_n such that w * w = e (the identity). In terms of permutations, this means the permutation is self-inverse.

### Product Interpretation
The product dKH_w * kH_x is computed in the Hecke algebra at q=1, resulting in a Laurent polynomial in q. The program reports matches where this product is non-zero and equal for pairs (x,y).

## Troubleshooting

### "w is not an involution"
This means the given Lehmer index does not correspond to a self-inverse permutation. Verify the index or check if you intended to use a different element.

### "Dual cache count does not match"
The dual cache was generated for a different n value. Ensure you're using the correct cache file.

### "Could not open dual_kl_bulk_n<n>.bin"
Generate the cache first using `dual_kl_bulk`, or specify the correct path with `--dual-bin`.

### "Failed to load KL data"
The KL data file is missing or corrupted. Regenerate using `kl_structure_bulk_opt_parallel`.
